# RAG 检索参数说明文档

## 概述

本系统的向量检索采用 **BGE-M3 双路混合检索**（密集向量 + 稀疏向量）策略。
以下三个参数控制检索的召回范围与融合方式，**按案件类型分为三组独立配置**。

---

## 参数详解

### 1. dense_top_k — 密集向量检索数量

| 属性 | 说明 |
|------|------|
| **含义** | BGE-M3 密集向量（语义编码）检索返回的候选文档数量 |
| **取值范围** | `3` ~ `30` |
| **影响** | 值越大，语义检索召回越全，但耗时增加、噪声也可能增多 |

### 2. sparse_top_k — 稀疏向量检索数量

| 属性 | 说明 |
|------|------|
| **含义** | BGE-M3 稀疏向量（词汇级匹配）检索返回的候选文档数量 |
| **取值范围** | `3` ~ `30` |
| **影响** | 值越大，关键词匹配越充分，但可能拉入不相关内容 |

### 3. alpha — 混合检索权重

| 属性 | 说明 |
|------|------|
| **含义** | 密集向量得分与稀疏向量得分的融合比例 |
| **取值范围** | `0.1` ~ `0.9`，步长 0.05 |
| **公式** | `最终得分 = alpha × 密集得分 + (1-alpha) × 稀疏得分` |
| **alpha > 0.5** | 偏向语义匹配（适合自然语言描述类问题） |
| **alpha < 0.5** | 偏向关键词匹配（适合有明确术语/法条引用的问题） |

---

## 各案件类型默认配置

### 民商事诉讼案件
| 参数 | 默认值 |
|------|--------|
| `dense_top_k` | 18 |
| `sparse_top_k` | 18 |
| `alpha` | 0.55 |

> 民商事案件覆盖面广，语义理解和关键词匹配并重，略偏语义。

### 刑事辩护案件
| 参数 | 默认值 |
|------|--------|
| `dense_top_k` | 22 |
| `sparse_top_k` | 18 |
| `alpha` | 0.35 |

> 刑事案件需大量召回参考案例，侧重关键词/罪名精确匹配。

### 行政诉讼案件
| 参数 | 默认值 |
|------|--------|
| `dense_top_k` | 18 |
| `sparse_top_k` | 18 |
| `alpha` | 0.40 |

> 行政案件侧重法条关键词匹配，兼顾语义理解。

## 前端参数页面

参数页面按三个案件类型分为三个标签页，可分别调整各类型的检索参数。
点击"恢复默认值"按钮可随时恢复以上默认配置。
调整后的参数保存在浏览器 localStorage 中，调用法律咨询接口时自动传入。

---

## 工作流程

```
用户问题
    │
    ├──→ 客户接待 Agent：分类案件类型，提取诉求和基本信息
    │
    ├──→ 律师 Agent：生成「1个主检索式 + 3-4个法律要素检索式」
    │
    ├──→ 按案件类型取对应的 dense_top_k / sparse_top_k / alpha
    │
    ├──→ 对每个检索式：
    │       ├── dense 向量检索 (召回 dense_top_k 条)
    │       └── sparse 向量检索 (召回 sparse_top_k 条)
    │
    ├──→ 多路结果合并去重
    ├──→ BGE Reranker 精排
    │
    └──→ 取 final_top_k 条作为最终参考文档
```

其中 `final_top_k` 由 LLM 根据案件复杂程度动态决定（一般为 5-8）。
